This Academy is meant to be **used, referenced, and built upon**. If it helped a
paper, a talk, a course, a README, or a blog post, a citation is welcome — and it
helps other developers find it.

To cite a **specific lesson**, add its title and URL — for example:
*Angolar, W. (2026). ping — ICMP Echo & Round-Trip Time. In The Ultimate Systems
Programming Academy.
https://angolardevops.github.io/systems-programming-academy/part-6-cli-tools/ping/*

## License & reuse

- **Example code** (everything under `examples/` in the repository) is released under the **MIT License** — use it freely, including commercially, keeping the copyright notice.
- **Lesson prose** is free to read, quote, and reference with attribution to the author. When quoting at length or translating, please link back to the source page.
- **The method is yours to adopt.** If you build your own byte-identical, cross-language teaching material inspired by this one, that's the highest compliment — a mention is appreciated, not required.

<Aside type="note" title="A permanent, versioned citation">
  For academic use, an archival DOI gives a citation that never rots. The
  recommended path is to connect the
  [GitHub repository](https://github.com/angolardevops/systems-programming-academy)
  to [Zenodo](https://zenodo.org/), which mints a versioned DOI on each release —
  the standard way to make a software/education artifact permanently citable. Until
  a DOI is minted, cite the URL above with an access date.
</Aside>
