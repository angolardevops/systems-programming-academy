## Não é um passeio pela sintaxe. É uma formação em engenharia.

A maioria dos tutoriais de linguagens para na sintaxe. Esta Academia ensina como
o software real é **desenhado, construído, testado, medido e colocado em
produção** — usando três linguagens em pontos diferentes do espectro da
programação de sistemas.

<CardGrid stagger>
  <Card title="Python" icon="seti:python">
    Legibilidade e velocidade. Seguimos a filosofia de Guido van Rossum: deve
    haver uma — e de preferência só uma — forma óbvia de o fazer.
  </Card>
  <Card title="Go" icon="seti:go">
    Simplicidade à escala. No espírito de Rob Pike: claro é melhor do que
    esperto; um pouco de duplicação é melhor do que uma dependência a mais.
  </Card>
  <Card title="Rust" icon="seti:rust">
    Segurança sem coletor de lixo. Na tradição de Graydon Hoare: o compilador é
    o seu par de programação.
  </Card>
</CardGrid>

## Cada lição é executável e testada

- ✅ **Nada de pseudocódigo.** Cada exemplo compila e corre.
- ✅ **Sem placeholders, sem TODOs.** O código vem com testes que pode correr.
- ✅ **Benchmarks reais.** Os números vêm de código que pode re-executar.
- ✅ **Bilingue.** Leia em Português ou English (troca no canto superior direito).

## Como a Academia está organizada

| Parte | Foco |
| --- | --- |
| **1 · Fundamentos** | Núcleo das linguagens: ownership, tipos, erros, generics, concorrência, testes — 20 lições nas três linguagens. |
| **2 · Engenharia Real** | Repository & DI, config, logging, caching, a pirâmide de testes, arquitetura hexagonal, CI/CD. |
| **3 · Automação DevOps** | Quatro ferramentas reais construídas e medidas de três formas: analisador de logs, exporter de métricas, gerador de config, agente de health-check. |
| **4 · Concorrência & Paralelismo** | Threads & estado partilhado, channels, I/O assíncrono — culminando num servidor web **mini-NGINX**. |
| **5 · Construir Frameworks** | Reconstruir as peças de uma web app: routing, um query builder, um DI container, validação, um template engine, um serializador JSON, e um test framework. |
| **Capstone** | Um guestbook seguro que derrota SQL injection e XSS, depois corre de verdade em HTTP + SQLite — fechado por uma comparação medida das três linguagens. |
