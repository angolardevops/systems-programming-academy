## Walter Angolar

**Systems engineer · Builder · Educator**

Walter Angolar is an Angolan software engineer focused on **systems
programming and infrastructure** — the layer where languages, operating
systems, and networks meet. His work follows a single conviction: the best
way to understand a system is to build it from first principles, test every
assumption, and measure the result.

## Trajectory

His path runs through the full stack of modern infrastructure engineering —
from years of hands-on DevOps and platform work to language design — and is
best told through what he has built and shipped:

- **Delonix Engine** — a production container platform written in Rust
  (10 crates): rootless containers, a single-ingress network model with
  private networks and internal DNS, image building, an operations console
  with a real terminal, and a KVM roadmap. Built solo, from `clone(2)` and
  network namespaces upward — the project that closes the gap between
  "understands containers" and "wrote a container engine".

- **Ayize** — a programming language (`.az`) with an interactive tutor,
  designed to lower the entry barrier into programming for Portuguese-speaking
  learners while staying honest about how computers actually work.

- **The Systems Programming Academy** — this site: a fully tested, bilingual
  (English/Português) curriculum teaching production software engineering in
  Python, Go, and Rust. Every example compiles, every claim is benchmarked on
  real hardware, and every project ships with its tests.

- **An engineering discipline built in public** — a six-month, full-time
  systems curriculum in Rust taken from ownership semantics to a shipping
  container engine; years of automating real operations (CI/CD pipelines,
  monitoring agents, infrastructure tooling) in Python and Go — the direct
  source of this academy's Part 3.

## Why this academy exists

Most programming material teaches syntax and stops where production begins.
Walter built the academy he wished existed: one where **every number is
measured** (not quoted), **every example runs** (not sketched), and the three
languages that power modern infrastructure are taught **against each other**,
so the trade-offs — runtime vs compiler, GIL vs scheduler, lock vs channel —
become visceral instead of theoretical.

The philosophy owes a debt to three designers whose languages anchor this
academy: **Guido van Rossum** (readability counts), **Rob Pike** (simplicity
is the art of hiding complexity), and **Graydon Hoare** (safety without
sacrifice).
