## Not a syntax tour. An engineering education.

Most language tutorials stop at syntax. This Academy teaches how real software is
**designed, built, tested, benchmarked, and shipped** — using three languages that
sit at different points on the systems-programming spectrum.

<CardGrid stagger>
  <Card title="Python" icon="seti:python">
    Readability and velocity. We follow the philosophy of Guido van Rossum:
    there should be one obvious way to do it.
  </Card>
  <Card title="Go" icon="seti:go">
    Simplicity at scale. In the spirit of Rob Pike: clear is better than clever;
    a little copying beats a little dependency.
  </Card>
  <Card title="Rust" icon="seti:rust">
    Safety without a garbage collector. In the tradition of Graydon Hoare:
    the compiler is your pair-programmer.
  </Card>
</CardGrid>

## Every lesson is executable and tested

- ✅ **No pseudo-code.** Every snippet compiles and runs.
- ✅ **No placeholders, no TODOs.** Code ships with tests you can run yourself.
- ✅ **Real benchmarks.** Numbers come from code you can re-run, not hand-waving.
- ✅ **Bilingual.** Read in English or Português (top-right language switch).

## How the Academy is organised

| Part | Focus |
| --- | --- |
| **1 · Foundations** | Language cores: memory, types, ownership, error handling, testing. |
| **2 · Real-World Engineering** | Architecture, CI/CD, observability, security — compared across languages. |
| **3 · DevOps Automation** | Build real tooling: CLIs, exporters, config generators. |
| **4 · Concurrency & Parallelism** | Threads, async, channels — culminating in a mini-NGINX. |
| **5 · Building Frameworks** | Rebuild an ORM, a web framework, and a green-thread runtime from scratch. |
| **Capstone** | A production microservice platform, no frameworks, each language where it fits. |
