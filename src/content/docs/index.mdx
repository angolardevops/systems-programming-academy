## Not a syntax tour. An engineering education.

Most language tutorials stop at syntax. This Academy teaches how real software is
**designed, built, tested, benchmarked, and shipped** — using three languages that
sit at different points on the systems-programming spectrum.

<CardGrid stagger>
  <Card title="Python" icon="seti:python">
    Readability and velocity. We follow the philosophy of Guido van Rossum:
    there should be one obvious way to do it.
  </Card>
  <Card title="Go" icon="seti:go">
    Simplicity at scale. In the spirit of Rob Pike: clear is better than clever;
    a little copying beats a little dependency.
  </Card>
  <Card title="Rust" icon="seti:rust">
    Safety without a garbage collector. In the tradition of Graydon Hoare:
    the compiler is your pair-programmer.
  </Card>
</CardGrid>

## Every lesson is executable and tested

- ✅ **No pseudo-code.** Every snippet compiles and runs.
- ✅ **No placeholders, no TODOs.** Code ships with tests you can run yourself.
- ✅ **Real benchmarks.** Numbers come from code you can re-run, not hand-waving.
- ✅ **Bilingual.** Read in English or Português (top-right language switch).

## How the Academy is organised

| Part | Focus |
| --- | --- |
| **1 · Foundations** | Language cores: ownership, types, error handling, generics, concurrency, testing — 20 lessons across all three languages. |
| **2 · Real-World Engineering** | Repository & DI, config, logging, caching, the testing pyramid, hexagonal architecture, CI/CD. |
| **3 · DevOps Automation** | Four real tools built and benchmarked three ways: a log analyzer, a metrics exporter, a config generator, a health-check agent. |
| **4 · Concurrency & Parallelism** | Threads & shared state, channels, async I/O — culminating in a **mini-NGINX** web server. |
| **5 · Building Frameworks** | Rebuild the pieces of a web app: routing, a query builder, a DI container, validation, a template engine, a JSON serializer, and a test framework. |
| **6 · Command-Line Tools & Dashboards** | Five elegant CLI tools from first principles: a `/proc` performance dashboard, a concurrent port scanner, `ping`, `traceroute`, and an eBPF latency profiler. |
| **7 · Build Your Own Interpreter** | Grow a character scanner into a Turing-complete language — lexer, parser, and tree-walking evaluator with variables, closures, recursion, conditionals, and boolean logic. |
| **Capstone** | A secure guestbook that defeats SQL-injection and XSS, then runs for real on HTTP + SQLite — closed by a measured comparison of the three languages. |
